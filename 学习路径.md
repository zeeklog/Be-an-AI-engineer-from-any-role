# AI 工程师学习路径

这份学习路径回答两个问题

- 要学什么
- 按什么顺序学

它综合了三类信息

- 仓库中的岗位数据分析 `人工智能工程岗位要求.md`
- AI 工程师角色分析 `角色/02-skills.md`
- 课程 [AI Engineering Buildcamp: From RAG to Agents](https://maven.com/alexey-grigorev/from-rag-to-agents)


## 如何使用这份学习路径

- 如果你还在选方向 可以先看根目录的 `README.md` 和 `人工智能工程岗位要求.md` 了解角色和市场
- 如果你已经有工程背景 可以直接跳到文末的 按角色出发路线
- 如果你想系统学习 可以按下面的阶段一步步来 不需要一次性掌握所有内容

整体目标是成为能够在生产环境中

- 设计和实现基于 LLM 的系统
- 上线并维护 RAG 和智能体应用
- 对系统的质量 成本 和可靠性负责


## 第一阶段 核心 AI 能力（1-2 个月）

这部分是 20% 的内容 覆盖 80% 的工作场景 也是岗位数据里最常被提到的能力

### 1 LLM 基础与 API 使用

- 理解 LLM 的能力边界 适合做什么 不适合做什么
- 使用 OpenAI Anthropic 等 API 调用模型
- 掌握常见的调用参数 温度 最大 tokens 系统提示等
- 学会结构化输出 让模型按照固定 JSON 或 Pydantic 模型返回结果
- 了解常见的安全问题 提示注入 敏感信息泄露 等

推荐练习

- 写一个命令行或小 Web 服务 封装一次 LLM 调用
- 让模型输出结构化 JSON 并在代码里做解析 校验和错误处理

### 2 Prompt 与上下文工程

- 基本 Prompt 模式 角色 定义 任务说明 few-shot 示例 约束等
- 为不同任务编写 Prompt 问答 总结 分类 结构化提取等
- 把 Prompt 抽象成模板 方便复用和版本管理
- 理解「Prompt 工程」正在演变为「上下文工程」

推荐练习

- 为同一个任务尝试多种 Prompt 写法 对比输出质量
- 设计一套可复用的 Prompt 模板 比如客服回复模板 报告生成模板

### 3 RAG 与搜索基础

- 理解 RAG 的核心思想 用自己的数据增强 LLM
- 理解文本检索与向量检索的区别和组合方式
- 掌握分块策略 针对长文档 API 文档 FAQ 等不同类型做切分
- 了解常见的向量数据库 Pinecone Weaviate Qdrant pgvector 等
- 知道如何从 PDF YouTube 文本 网站等来源抽取和清洗数据

推荐练习

- 做一个 FAQ 助手 从一组 FAQ 文档中回答问题
- 做一个文档问答系统 支持上传文档 并基于文档回答问题

### 4 智能体 Agents 与工具调用

- 理解函数调用和工具调用模式 让 LLM 能够调用外部 API
- 理解「观察-思考-行动」循环 如何设计智能体的主循环
- 学会使用一个主流框架 LangChain LangGraph PydanticAI OpenAI Agents SDK 等
- 了解 Model Context Protocol MCP 这类标准工具协议
- 初步接触多代理模式 路由流水线 协调等

推荐练习

- 写一个网页研究助手 自动搜索 并整理多篇网页的观点
- 做一个简单的数据提取流水线 比如从网页批量抽取结构化信息

### 5 测试 评估 与成本意识

- 学会为 LLM 应用写测试 不仅是单元测试 还包括评估脚本
- 理解 LLM-as-judge 的思路 用一个模型来评估另一个模型的输出
- 为常见任务设计打分标准 精确度 覆盖度 风格 一致性等
- 记录与监控调用成本 tokens 消耗和延迟

推荐练习

- 为自己的 FAQ 或 RAG 项目写一套小型评估脚本
- 采样一批真实问题 人工标注基础答案 再用 LLM 打分并与人工结果对比


## 第二阶段 工程与基础设施（1-3 个月）

岗位数据表明 93% 以上的 AI 工程岗位要求 GenAI 以外的工程能力 这些能力决定你能否把 Demo 变成生产系统

### 6 Python 与软件工程基础

- 扎实掌握 Python 语法 类型提示 常用标准库
- 掌握测试框架 pytest 等
- 了解代码质量工具 格式化 静态检查 等
- 熟悉 Git 工作流 分支 提交 代码评审

推荐练习

- 用清晰的项目结构重构自己的小项目 把 notebook 拆成模块
- 为项目添加测试 让主要逻辑可被自动化验证

### 7 Web 与 API 开发

- 熟悉一个 Python Web 框架 推荐 FastAPI
- 理解 REST API 的设计原则 基本认证和限流等
- 如果你有前端背景 可以进一步学习 React 或 Next.js 做全栈应用

推荐练习

- 把已有的 RAG 或 Agent 项目包装成 HTTP API
- 做一个简单前端 调用自己的 AI API 展示结果

### 8 云平台 与 DevOps

- 至少熟悉一个云平台 AWS Azure GCP
- 学会使用 Docker 进行容器化 构建镜像 运行容器
- 了解基本的部署方式 如容器服务 函数计算 传统虚机等
- 理解 CI/CD 的核心概念 自动测试 自动部署

推荐练习

- 为自己的项目写 Dockerfile 本地用 docker compose 运行
- 在任一云平台上完成一次从代码到可访问服务的端到端部署

### 9 数据库 与存储

- 掌握 PostgreSQL 的基础使用 表 关联 查询 索引等
- 理解向量数据库的使用模式 嵌入存储 检索 筛选和排序
- 了解 Redis 作为缓存和会话存储的常见用法

推荐练习

- 用 PostgreSQL 或 pgvector 实现一个简化版的 RAG 系统
- 为项目增加缓存层 降低对 LLM 的重复调用

### 10 数据工程 与 ML 基础

- 理解数据管道的基本概念 抽取 转换 加载
- 了解任务调度工具 Airflow Spark 等在企业内的使用方式
- 掌握 embeddings 的概念 和简单的模型评估指标
- 对 PyTorch 有一个实用层面的认识 方便看懂项目代码

推荐练习

- 为自己的项目实现一个简单的定时数据更新流水线
- 用 PyTorch 或现成的库训练一个小模型 并与直接调用 API 的方案对比


## 第三阶段 监控 评估 与生产化（1-2 个月）

这部分是很多候选人忽略 但在岗位里权重越来越高的能力

### 11 监控 与可观测性

- 理解日志和 Trace 的区别和作用
- 了解 OpenTelemetry 这类标准 如何在项目中打点
- 会为关键路径加上监控和告警
- 能够搭建简单的可视化面板 比如基于 Grafana

推荐练习

- 为自己的一个服务增加请求级别的日志和简单的指标统计
- 把这些数据发送到一个可视化工具中查看

### 12 评估体系 与 Guardrails

- 能够设计离线评估流程 回放历史请求做回归测试
- 了解合成数据在评估中的作用与局限
- 理解 Guardrails 的概念 输入过滤 输出约束 安全策略等
- 知道在哪些场景必须进行人工审核和多级防护

推荐练习

- 为已有项目加上一道安全过滤层 对输出做基本检查
- 设计一个简单的回归测试集 每次改 Prompt 或模型前后都跑一遍

### 13 从 Demo 到生产

- 能够把 notebook 中的探索性代码拆成明确的模块
- 会写最小可行的设计文档 说明需求 边界 失败模式和监控
- 掌握灰度发布和回滚的基础思路
- 理解在真实场景中如何与产品 业务和安全团队协作

推荐练习

- 选一个自己做过的 Demo 按「生产项目」的要求重新做一次
- 写清楚需求 方案 评估方式 监控点和上线策略


## 技能优先级速查表

结合岗位数据和本仓库中的分析 可以把技能大致分成三档

### 必须具备

- Python
- Prompt 工程与上下文工程
- RAG 模式理解
- 至少一个主流云平台 AWS Azure GCP 之一
- Docker

### 高价值补充

- LangChain 或 PydanticAI 等 AI 编排框架
- TypeScript
- FastAPI
- Kubernetes
- CI/CD
- PyTorch 基础

### 差异化能力

- 高级 Agent 框架 LangGraph CrewAI 等
- 模型微调 和特定垂类的定制模型
- 专门的评估框架和实践经验
- 向量数据库 Pinecone Weaviate Qdrant 等的深度使用经验
- 多智能体模式 和复杂工作流编排


## 典型 AI 工程技术栈

- 应用层 React Next.js FastAPI
- AI 编排层 LangChain LangGraph PydanticAI
- 模型层 OpenAI Anthropic Groq 或本地模型
- 检索层 向量数据库 Pinecone Weaviate Qdrant pgvector
- 基础设施 Docker Kubernetes AWS GCP Azure
- 监控与日志 Logfire Grafana OpenTelemetry
- 评估 LLM-as-judge 专用评估框架 如 Evidently 等
